hasMeta (any)
Checks for a __type metamethod. It can be either a function or a string. If it is a function, it will be called with the value as an argument.
As hasMeta allows you to have a function be called with the value, the type returned can depend on the value. This is called "dynamic types" in ltypekit. It means that you can define a single type (Boolean) and have it return two types (True or False).

Parameters:

  • any any value

Returns:

    string or false

Tables

baseTypes
Native type checking. Direct type replacement and extension functions. This is done so native types match the Uppercase style.

Fields:

  • number "Number"
  • string "String"
  • function "Function"
  • thread "Thread"
  • nil "Nil"
  • userdata "Userdata"
  • table "Table"
  • boolean "Boolean"
nativeResolvers
Type resolvers for native types. Each of the functions are prefixed with "is" and suffixed with an uppercase type name.

Fields:

  • isNumber Returns either "Number" or false.
  • isString Returns either "String" or false.
  • isFunction Returns either "Function" or false.
  • isThread Returns either "Thread" or false.
  • isNil Returns either "Nil" or false.
  • isUserdata Returns either "Userdata" or false.
  • isTable Returns either "Table" or false.
  • isBoolean Returns either "Boolean" or false.
typeof
Returns the type of a value. It requires to be a table with methods to exploit the mutability of Lua tables.

Fields:

  • resolvers List of resolvers used by typeof.
  • add Adds a new type.
kindof
Returns the kind of a value. It is a table for the same reason than typeof.

Fields:

  • resolvers List of resolvers used by kindof.
